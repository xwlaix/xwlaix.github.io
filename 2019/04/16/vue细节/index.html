vue细节

发表于 2019-04-16 | 分类于 记录

  1. 动态属性名：可使用表达式来设置动态属性或方法名：

  2. computed/methods/watch

    • computed可使用get/set
    • computed 可缓存，但不可传参，会根据data中的属性变化而变化，即是根据响应式依赖来变化，而Date不是响应式依赖，即不会变化；method则每次都会进行计算，但可传参。
    • watch用于处理异步或开销比较大的操作，如输入搜索时。

  3. style绑定

    • 直接对象或变量对象
    • 计算属性
    • 直接style或style对象
      <div :style="{'color':'red',fontSize:fontSize+'px'}样式</div>"
    • 数组结合三目/数组结合对象

  4. v-if条件渲染

    • 可使用template包裹元素，template会被当成不可见的包裹元素
    • 多条件判断 v-if、v-else-if、v-else
  5. 添加key防止vue重复利用不想被重复利用的元素。
  6. v-if和v-show
    1. v-if是组建的销毁和重建，初始条件为false，则什么都不做。
    2. v-show是display:none和block之间css切换，基于渲染，不管初始条件都会渲染
  7. v-for
    1. 可使用in或者of
    2. 也可遍历对象：v-for="{value,key,index} in obj"
    3. 也根据template渲染多个组合元素
  8. v-for和v-if
    1. v-for优先级更高，所以v-if会重复运行于每个v-for循环中，所以尽量不要一起使用，可先使用计算属性对数据进行过滤再遍历。

  9. 更改响应式数据

    1. Vue.set(object,key,value)
    2. this.$set(object,key,value)
    3. this.item.splice(index,1,newValue)
    4. 批量添加属性：

  10. 事件修饰符

    1. .passive:滚动的默认事件会立即触发，即告诉浏览器不想组织事件默认的触发，可提升移动端性能<div :scroll.passive="onScroll/>"
    2. .capture:添加事件监听器时使用事件捕获模式，即元素自身触发的事件现在此处理，然后才交由内部元素进行处理
    3. .self:只当在event.target是当前元素自身时触发处理函数，即事件不是从内部元素触发的
    4. .once:点击事件只会触发一次
    5. 键盘修饰符：<input v-on:keyup.enter="submit">

  11. v-model

    1. 选择框

    2. 修饰符.lazy:在change时而非input时更新<input v-model.lazy="msg">

      chang事件是在input失去焦点时触发，input事件是在value值变化时触发，但脚本改变value值不会触发

    3. 修饰符.number:输入值转为数值
    4. 修饰符.trim:过滤收尾空白字符
  12. Prop
    1. 使用v-bind=”obj”会将对象的每个属性都作为一个独立的prop传入进去，所以接受时也需要逐个属性接收。
    2. props虽然是单向数据流，但对于引用类型，子组件还是可以改变父组件的状态。
    3. props会在组件实例创建之前进行验证，所以实例的属性再default或validator中是不可用的。
  13. 自定义事件
    1. 自定义事件需要注意事件名为小写或-分隔，因为$emit(‘baseEvent’)虽然事件名不会变，但在html中改事件名会被转化为小写，不会监听到。

  14. slot

    1. 具名插槽

    2. 作用域插槽

  15. 组件通信

    1. vuex/eventBus
    2. prop/$emit
    3. $children/$parent
    4. provide/inject
    5. $refs

  16. scope

    1. socped属性会自动添加一个唯一的属性（比如data-v-21e5b78）为组件内css指定作用域，编译的时候.list-container:hover会被编译成类似.list-container[data-v-21e5b78]:hover

  17. 路由

    1. 区分：this.$router指路由器，this.$router指当前路由

    2. 通配符：捕获所有路由或404 Not found路由

    3. 当使用一个通配符时，$route.params内会自动添加一个名为pathMatch参数。它包含URL通过通配符被匹配的部分：

    4. 点击等同于调用router.push(…)方法，因为会在内部调用该方法，进而在history栈添加一个新的记录

    5. 使用push时，如果提供的path不完整，则params会被忽略，需要提供路由的name或手写完整的带有参数的path：

    6. router.push/router.replace/router.go效仿于window.history.pushState/window.history.replaceState/window.history.go

    7. 命名识图：router-view可设置名字，如果router-view没有设置名字，那么默认为default

    8. 路由使用props：可将路由参数设置为组件属性

    9. beforeRouteEnter：可使用beforeRouteEnter来提前获取接口数据，同时需要在next后才能访问到实例：

    10. 路由设置有参数时，如果跳转页面后再通过返回时，路由会保留有参数，如果通过push跳转返回，则不会保留该参数，这在第三方调用模块传参是需要注意。

  18. loader

    1. Vue Loader编译单文件的template块时，会将所有遇到的URL资源转为webpack模块请求：

    2. 资源URL转换规则

      1. 如果是绝对路径，例如/images/foo.png,则会原样保留。
      2. 如果以.开头，将会被看作相对模块依赖，并按照你的本地文件系统上的目录结构进行解析。
      3. 如果路径以-开头，其后部分将会被看作模块依赖。
      4. 如果路径以@开头，也会被看作模块依赖。
